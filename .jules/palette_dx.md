## 2024-06-04 - Narrowing `mixed` array structures in private methods
**Learning:** Found a recurring pattern of "mystery meat" arrays passed as `mixed` which required runtime `is_array()` checks at the top of the method. For example, `mixed $formsData` in `FormAssertionsTrait`. Leaving them as `mixed` bypassed static analysis and caused cognitive load as the dev has no clue it actually holds an array of arrays representing form trees.
**Action:** Always identify methods that take `mixed` but immediately guard with `is_array()`. Narrow the parameter to `array` and add a PHPDoc structure (like `@param array<string, mixed>`) to describe the contents. Then strictly type-hint the callers if they dynamically extract this array. This drops the cognitive load of what data is expected and ensures PHPStan checks correctness.
