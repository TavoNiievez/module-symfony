## 2026-04-16 - [Router Assertions Trait O(1) Optimization]
**Learning:** O(N) iteration inside `RouterAssertionsTrait::findRouteByActionOrFail` with `str_ends_with` was a performance bottleneck that was solved by implementing a two-level cache. We memoize the action lookup itself in `$this->cachedRoutesByAction[$action]`. Also, array initialization must guard against redefining `$this->cachedRoutesByAction` ensuring the first controller takes precedence when there are multiple routes for a single controller. It is critical to manually nullify these cached values in testing teardowns (`tearDown` and `_after`) to prevent memory leaks in continuous CI testing environments.
**Action:** Always verify if an O(N) lookup in an assertion method can be replaced by a one-time array build and O(1) hash map access. Furthermore, explicitly clear testing state (e.g., caches, maps) in lifecycle hooks.
