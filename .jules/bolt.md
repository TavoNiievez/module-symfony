\n## 2024-XX-XX - Session Assertions Optimization\n**Learning:** Iterative methods in assertion traits (e.g., `seeSessionHasValues`, `logoutProgrammatically`) can suffer from micro-inefficiencies like repeated Dependency Injection container lookups (`getCurrentSession()`) and dynamic array instantiations within loops.\n**Action:** Hoisted the session retrieval out of the loop and replaced `in_array()` with dynamically constructed arrays with strict `===` checks and logical `||` operators to avoid memory allocation and O(N) lookups inside loops.
